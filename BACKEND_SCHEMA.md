## 1. Database Architecture Overview

### Why Firebase Firestore?

```
✅ Real-time updates (great for live notifications)
✅ Automatic scaling (handles 1M+ documents)
✅ Built-in authentication (Firebase Auth)
✅ Global distribution (low latency worldwide)
✅ No server management (serverless)
✅ ACID transactions (consistency guaranteed)
✅ Native offline support (mobile-friendly)
✅ Excellent for startup phase (pay-as-you-go)
```

### 2.1 `users` Collection

**Purpose:** Store user accounts, profiles, and settings

```javascript
Collection Path: /users
Document ID: {userId} (Firebase UID)

Example Document ID:
"abc123xyz789def456"

Document Structure:
{
  // Authentication
  userId: string (auto-generated by Firebase Auth),
  phone: string (unique, indexed),
  phoneVerified: boolean,
  
  // Profile Information
  name: string (required, indexed),
  email: string (optional),
  profileImage: string (Firebase Storage URL),
  bio: string (optional, max 200 chars),
  
  // Account Type
  userType: enum {
    "tenant",    // Looking for rental
    "owner",     // Listing properties
    "admin"      // Platform admin
  },
  
  // Account Status
  status: enum {
    "active",       // Can use app normally
    "suspended",    // Blocked by admin
    "deleted"       // Soft delete (keep for audit)
  },
  
  // Settings & Preferences
  language: enum {
    "en",  // English
    "ta"   // Tamil
  },
  
  notifications: {
    email: boolean,
    sms: boolean,
    push: boolean,
    paymentReceipts: boolean,
    propertyAlerts: boolean
  },
  
  // Privacy & Security
  privacyLevel: enum {
    "public",      // Profile visible to all
    "private",     // Profile hidden
    "contacts_only" // Visible to contacts only
  },
  
  blockedUsers: array of strings (userIds),
  
  // Analytics & Usage
  totalUnlocks: number (count of payments made),
  totalMoneySpent: number (in rupees),
  totalPropertiesListed: number (for owners),
  totalPropertiesViewed: number,

  referralCode: string (unique, optional),
  referredBy: string (userId of referrer, optional),

  lastLoginAt: timestamp,
  lastActivityAt: timestamp,
  
  // Metadata
  createdAt: timestamp,
  updatedAt: timestamp,
  deletedAt: timestamp (optional, for soft deletes)
}

Indexes:
- phone (unique)
- userType
- status
- createdAt
- email (sparse, optional)
```

**Subcollections:**

### 2.3 `payments` Collection

**Purpose:** Store all payment transactions

```javascript
Collection Path: /payments
Document ID: {paymentId} (formatted: userId_propertyId_timestamp)

Example:
"abc123_prop001_1623456789"

Document Structure:
{
  // Identifiers
  paymentId: string (unique),
  transactionId: string (unique, indexed),
  
  // References
  userId: string (indexed),
  propertyId: string (indexed),
  ownerId: string (owner of the property, indexed),
  
  // Amount
  amount: number (in rupees),
  currency: string ("INR"),
  
  // Payment Gateway
  gateway: enum {
    "cashfree",
    "razorpay",
    "stripe"
  },
  
  // Gateway-Specific Info
  gatewayOrderId: string (merchant order ID),
  gatewayPaymentId: string (transaction ID from gateway),
  gatewayCustomerId: string (customer ID from gateway),
  
  // Payment Method
  paymentMethod: enum {
    "upi",        // UPI apps
    "card",       // Credit/Debit card
    "netbanking", // Internet banking
    "wallet"      // Wallets (Google Pay, etc.)
  },
  
  // Card Details (if applicable)
  cardDetails: {
    last4: string (last 4 digits),
    brand: enum {
      "visa",
      "mastercard",
      "amex"
    },
    expiryMonth: number,
    expiryYear: number
  },
  
  // UPI Details (if applicable)
  upiDetails: {
    upiId: string (masked),
    vpa: string (Virtual Payment Address, optional)
  },
  
  // Status
  status: enum {
    "pending",      // Awaiting gateway response
    "completed",    // Successful payment
    "failed",       // Payment declined
    "cancelled",    // User cancelled
    "refunded",     // Refunded to user
    "expired"       // Timeout, payment lapsed
  },
  
  gatewayStatus: string (raw status from gateway),
  
  // Security & Verification
  signatureVerification: {
    isVerified: boolean,
    verifiedAt: timestamp,
    verificationMethod: enum {
      "hmac",      // HMAC-SHA256 signature
      "polling",   // Verified via polling
      "webhook"    // Verified via webhook
    },
    verificationDetails: object (for debugging)
  },
  
  // Refund Details (if applicable)
  refund: {
    refunded: boolean,
    refundAmount: number,
    refundId: string (gateway refund ID),
    refundReason: string,
    refundedAt: timestamp,
    refundStatus: enum {
      "pending",
      "completed",
      "failed"
    }
  },
  
  // Contact Info (Denormalized, unlocked data)
  unlockedContact: {
    ownerName: string,
    ownerPhone: string,
    ownerEmail: string,
    ownerProfileUrl: string (optional)
  },
  
  // Property Info (Snapshot for audit trail)
  propertySnapshot: {
    propertyId: string,
    propertyTitle: string,
    propertyType: string,
    location: {
      city: string,
      area: string
    },
    monthlyRent: number,
    bedrooms: number
  },
  
  // Error Handling
  errorDetails: {
    error: boolean,
    errorCode: string (optional),
    errorMessage: string (optional),
    errorAt: timestamp
  },
  
  // Metadata
  createdAt: timestamp,
  updatedAt: timestamp,
  completedAt: timestamp (when status changed to "completed"),
  
  // IP & Device Info
  ipAddress: string (for fraud detection),
  userAgent: string (device info),
  
  // Audit Trail
  auditLog: array of objects {
    action: string,
    status: string,
    timestamp: timestamp,
    notes: string
  }
}

Indexes:
- userId
- propertyId
- transactionId (unique)
- status
- gateway
- createdAt
- completedAt
```

## 3. Firestore Security Rules

```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    
    // Users collection
    match /users/{userId} {
      // Own profile: full access
      allow read, write: if request.auth.uid == userId;
      
      // Others: read public info only
      allow read: if request.auth != null 
        && resource.data.privacyLevel == 'public';
      
      // Create new user (signup)
      allow create: if request.auth.uid == userId
        && request.resource.data.userId == userId;
    }
    
    // Properties collection
    match /properties/{propertyId} {
      // Anyone can read active properties
      allow read: if resource.data.status == 'active';
      
      // Owner can read their own properties
      allow read: if request.auth.uid == resource.data.ownerId;
      
      // Owner can create/update their own properties
      allow create, update: if request.auth.uid == get(/databases/$(database)/documents/users/$(request.auth.uid)).data.userType == 'owner';
      
      allow delete: if request.auth.uid == resource.data.ownerId;
    }
    
    // Payments collection
    match /payments/{paymentId} {
      // User can read their own payments
      allow read: if request.auth.uid == resource.data.userId;
      
      // Owner can read payments for their properties
      allow read: if request.auth.uid == resource.data.ownerId;
      
      // Only backend (via service account) can write
      allow create, update: if request.auth.uid == null 
        && request.auth.token.iss == "https://securetoken.google.com/your-project";
    }
    
    // Unlocks collection
    match /unlocks/{unlockId} {
      // User can read their own unlocks
      allow read: if request.auth.uid == resource.data.userId;
    }
    
    // Saves collection
    match /saves/{saveId} {
      // User can read, create, update, delete their own saves
      allow read, write: if request.auth.uid == resource.data.userId;
      
      // Allow create without reading (for new saves)
      allow create: if request.auth.uid == request.resource.data.userId;
    }
  }
}
```

## 5. Database Queries

### Common Queries

```javascript
// 1. Get user profile
const userDoc = await db.collection('users').doc(userId).get();

// 2. Get all active properties in a city
const properties = await db.collection('properties')
  .where('location.city', '==', 'Thanjavur')
  .where('status', '==', 'active')
  .orderBy('createdAt', 'desc')
  .limit(20)
  .get();

// 3. Get user's payment history
const payments = await db.collection('payments')
  .where('userId', '==', userId)
  .orderBy('createdAt', 'desc')
  .get();

// 4. Check if user unlocked a property
const unlock = await db.collection('unlocks')
  .doc(`${userId}_${propertyId}`)
  .get();

// 5. Get user's saved properties
const saves = await db.collection('saves')
  .where('userId', '==', userId)
  .get();

// 6. Get owner's property inquiries
const unlocks = await db.collection('unlocks')
  .where('propertyId', '==', propertyId)
  .get();

// 7. Verify payment signature (server-side)
const payment = await db.collection('payments')
  .where('gatewayOrderId', '==', orderId)
  .get();

// 8. Get nearby properties (geo-query)
const nearby = await db.collection('properties')
  .where('location.geoPoint', '<=', geoPoint + radius)
  .where('location.geoPoint', '>=', geoPoint - radius)
  .get();
```

## 9. Performance Optimization

### Database Optimization Tips

```
1. Denormalization Strategy
   - Store owner info in properties collection
   - Store property snapshot in payments
   - Reduces joins/reads

2. Collection Limits
   - Each collection: max 10,000 documents for free tier
   - Scale planning: 1M properties = 100+ projects

3. Query Optimization
   - Always use indexed fields in WHERE clauses
   - Limit result sets (pagination)
   - Use order limiting instead of client-side sorting

4. Caching
   - Cache user profile in context
   - Cache property listings in React Query
   - Firestore client-side caching (offline support)

5. Batch Operations
   - Batch writes (up to 500 docs per batch)
   - Reduces latency for multi-document operations
```

## 12. Data Retention & Privacy

### Data Retention Policy

```
Active User Data:
- Retained indefinitely (or until user deletes account)

Deleted User Data:
- Soft delete: keep for 30 days (audit trail)
- Hard delete after 30 days
- Anonymize personal data

Payment Records:
- Keep for 7 years (legal requirement - PCI-DSS)

Logs & Analytics:
- Keep for 90 days
- Aggregate and archive after 90 days
```
